When would you want to use a remote repository rather than keeping all your work local?
To share your repository with other people or use it from other computer

Why might you want to always pull changes manually rather than having Git automatically stay up-to-date with your remote repository?
Because you may not want to share all your changes to your remote repository

Describe the differences between forks, clones, and branches. When would you use one instead of another?
Branches are only in local, forks are only in GitHub and clone may be used for a local repository or for a remote one.
If you want to make changes on a repository of GitHub and you don't have permission, you can fork it and the clone it into your local repository, and then modify it and push your changes to your forked version.
